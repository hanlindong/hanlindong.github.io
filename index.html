iOS main函数汇编

本文初步涉及汇编相关：

0x2730: pushl $0 （ 将出错码入栈）

当异常发生时，如果控制单元没有自动地把一个硬件错误代码插入到栈中，相应的汇编语言片段会包含一条pushl $0指令，在栈中垫上一个空值,如果错误码已经被压入堆栈,则没有这条指令。然后，把异常处理函数的地址压进栈中；函数的名字由异常处理程序名与do_前缀组成

0x2732: movl %esp, %ebp

栈指址（低地址）%esp栈顶值赋给栈基址（高地址）%ebp栈底，设置 main函数的栈基址

0x2734: andl $-16, %esp

等价于 andl $0xfffffff0,%esp,也就是说把%esp内的最后4位给抹掉，强制让%esp的值是16的倍数，GCC默认的堆栈是16字节对齐的，为了加快CPU的访问效率

OS X 中的 Quick Look 文件预览

QuickLook（快速预览）首次出现在 Mac OS X 10.5 中，这个功能让用户可以在不打开相应的软件的情况下，按下空格键即可快速预览文件内容，这个 Mac OS X 特性，因为它的简单和高效使得这个功能被使用很广泛。

一般来说，Quick Look 默认只支持少数的一些文件类型，也可以通过安装插件（譬如说 QLMarkdown 用来预览 md 文件）增加文件预览类型。

PS 与 Sketch

用过一段时间 Sketch 后，重新看去年年底在中华女子学院宣传时设计的 Logo，发现其实两款软件各有所长，谁也取代不了谁。

当时在 Mac AppStore 购买的 Sketch 现在看来一点也不贵，因为毕竟生产力上去了，尤其是光栅化预览。可以直接看到矢量图像像素化后的效果。Sketch的光栅化引擎相当出色，特别是字体边缘渲染干净利落，而AI转存的图像，边缘的抗锯齿不太理想，且难以控制。可能是我AI用的少，没有把功能吃透。但至少像素级预览绝对是设计师做矢量图像素级调整的有力助手。

灵活的切图和输出功能，而且直接输出两倍大的Retina图，还能统一加上后缀。PS和AI的切片输出貌似还是停留在Web应用的时代。

锚点对齐到像素，不知道省了我多少微调的时间。

而 PS 我也一直在订阅，最近却不怎么用了，虽然说 Photoshop 90%的功能Sketch都做不到，可能是今年一直没画复杂的图，用不上那么多的高级功能。

高级用法1：多线路规则

特点：多条线路使用同一规则，免配置，自由切换

思路：

Proxy1|

Proxy2| … + Rule

Proxy3|

上面Proxy每一个序号都代表了一条线路，也就是说如果有多条线路，只需要一个规则即可完成瞬间切换，后期也不需要更新规则后再次配置线路信息。

优点：

1.下载规则免配置;

番号其实就是一种识别码，是一种用于表示电影类型的序列号。利用搜索引擎对番号进行搜索，就可以获取电影的磁力链接，进而利用下载工具下载电影或在线观看电影。
