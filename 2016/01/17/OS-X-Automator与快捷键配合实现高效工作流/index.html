OS X Automator与快捷键配合实现高效工作流

  • OSX
  • 效率

接上文：『基于Node-js的Github-Pages博客系统』

上文介绍了个人博客的搭建，这篇文章我们来谈一谈关于 OS X 下自带应用程序「Automator」配合「终端」及剪切板工具「CopyLess」的使用来实现高效工作流。

同步到个人博客的步骤较为繁琐，我们先来看一下：

  • 开启终端

  • 输入 cd /Users/ YourName /Documents/hexo

  • 输入 hexo g

  • 输入 hexo d

每次都输入这么多十分让人头疼，接下来我们进行简化。

Automator：

1、打开自带的Automator（可以用右上角Spotlight搜索找到）

2、在创建界面选择“服务”

3、如图：

左侧列表中，找到：操作——资源库——实用工具——开启应用程序

右侧选择“没有输入”

最后双击“开启应用程序”

在新出现的界面里选择需要的应用程序，这里选择了“终端”（如果列表中没有，可以在列表最下方的“其他”找到你的所有软件）

4、按 Command+Shift+Option+S 随便起个你能记得住的名字。这里起名“开启终端”

系统偏好设置 - 键盘 - 快捷键 - 服务

1、找到刚才起名的“开启终端”

2、在这里把前面的对号选中，然后后面自行设置快捷键。

请尽量设置三个以上键位的快捷键，并且避免与常用快捷键重复（与其他快捷键重复会导致设置无效，请重新设置）。这里使用了 Command+Option+0 ，之后即可通过此快捷键开启终端。

开启CopyLess快捷键

1、下载CopyLess软件，AppIcon如图：

拷贝:

cd /Users/ YourName /Documents/hexo
hexo g
hexo d

2、打开CopyLess，最上方即你所拷贝的文字，点击右方将其固定：

3、打开Favorites，复制的文字的快捷键为Control+Option+Command+1。

现在可以用快捷键开启终端并直接同步文章至你的博客

仅需两步：

1.Command+Option+0

2.Control+Option+Command+1
